Pokémon data warehousing & analytics pipeline · 2025

Started with a simple question any competitive Pokémon player has asked — which generation actually produces the strongest Pokémon? What followed was a full data engineering pipeline built from scratch to find out.

Raw Pokémon data was gathered through web scraping, intentionally targeting messy, imperfect datasets full of anomalies to simulate real-world data engineering challenges. The database was modeled using Lucidchart ERDs, raw files were loaded into Google Cloud Storage, and BigQuery was used to build a multi-stage ETL pipeline moving data from raw → staging → intermediate → mart layers. VertexAI enriched the tables with additional context, SQL and Python handled data transformations, and the entire pipeline was automated using dbt — keeping everything reproducible, versioned, and clean.
